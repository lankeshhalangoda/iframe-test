Survey Embed Test

Waiting for survey completion...
Current test: when the survey submit button is clicked inside the iframe, the iframe sends a message to this parent page using postMessage.

How this was done

1. What we wanted to achieve

We had a survey loaded inside an iframe. When the survey is submitted, we wanted the parent application to know that the survey was completed.

Since the survey page and the parent app are on different domains, the correct way to communicate is by using window.parent.postMessage(...).

2. Setup used in this test

  • Survey URL: https://emojot.com/iframetestlnksh
  • Parent app URL: https://iframetest-lnksh.vercel.app
  • Trigger used: click on the survey button with class .btn-submit
  • Message type used: SURVEY_COMPLETED

3. How the flow works in simple terms

  1. The parent app loads the survey inside an iframe.
  2. The survey page listens for a click on .btn-submit.
  3. When clicked, the survey sends a message to the parent page.
  4. The parent page listens for that message.
  5. When it receives the expected message from the correct origin, it updates the UI and URL.

4. Parent page code

This is the main page hosted on Vercel. It loads the iframe and listens for the completion signal.

Parent page listener example 
<!-- Status shown to the user -->
<div id="status">Waiting for survey completion...</div>

<!-- Survey loaded inside an iframe -->
<iframe
  src="https://emojot.com/iframetestlnksh"
  title="Survey"
  width="100%"
  height="700"
></iframe>

<script>
  // Listen for messages coming from the iframe
  window.addEventListener("message", function (event) {
    console.log("message received", event.origin, event.data);

    // Security check:
    // only accept messages from the survey domain
    if (event.origin !== "https://emojot.com") return;

    // Check that this is the message we expect
    if (event.data && event.data.type === "SURVEY_COMPLETED") {
      // Update visible status on the page
      document.getElementById("status").textContent =
        "Survey completed signal received";

      // Update the parent URL without reloading the page
      const url = new URL(window.location.href);
      url.searchParams.set("surveyCompleted", "true");
      url.searchParams.set("surveyId", event.data.surveyId || "unknown");
      window.history.replaceState({}, "", url);
    }
  });
</script>

5. Survey page code

This script is added to the survey page. It listens for the submit button click and sends the message to the parent.

Survey page script example 
<script>
  (function () {
    // Optional safety check so this only runs on the intended survey path
    if (window.location.pathname !== "/iframetestlnksh") return;

    // Listen for any click in the page
    document.addEventListener("click", function (event) {
      // Check whether the clicked element is the submit button
      const btn = event.target.closest(".btn-submit");
      if (!btn) return;

      // Small delay so the survey's own click logic can run first
      setTimeout(function () {
        // Send a message from the iframe to the parent page
        window.parent.postMessage(
          {
            type: "SURVEY_COMPLETED",
            surveyId: "iframetestlnksh",
            completed: true,
            submittedAt: new Date().toISOString()
          },
          "https://iframetest-lnksh.vercel.app"
        );
      }, 150);
    });
  })();
</script>

6. Why this did not break the survey

  • It only listens for clicks; it does not interfere with normal behavior.
  • It does not use preventDefault().
  • It does not stop propagation.
  • It only sends a passive message to the parent page.
  • The delay allows the survey’s normal submit logic to run first.

7. Important note for production

In this test, the signal is sent on button click. That proves communication works. For production, it is better to send the signal only after the survey is actually submitted successfully, not only when the button is clicked.

8. Common issues to watch for

  • The iframe may be blocked by X-Frame-Options or frame-ancestors.
  • The target origin in postMessage must match the parent URL exactly.
  • The parent page must validate event.origin.
  • If .btn-submit changes, the selector must be updated.
